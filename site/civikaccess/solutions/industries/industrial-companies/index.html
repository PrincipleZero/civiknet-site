FAQ

How secure is CivikNet?

CivikNet is single tenant, SOC2 and ISO 27001 certified, and GDPR compliant — with additional alignment to NIST and FERPA standards for public-service deployments. All data is encrypted at rest with AES 256 and in transit with TLS 1.2+. The platform follows data privacy regulations and guidelines to protect each individual user. Enterprise users can be authenticated with Single-Sign-On (SSO). Integrated systems can mirror the underlying system permissions.

Which integrations does CivikNet Agents have?

Enterprise customers can use a wide range of off-the-shelf connectors. New connectors are added and included in the offering. Free tier users can currently add integrations to Google Drive, Sharepoint, Google Calendar, Outlook Calendar. Enterprise customers can also build custom integrations to CivikNet Agents. CivikNet Agents supports meeting recordings in Zoom, Microsoft Teams, and Google Meet for both tiers.

Can CivikNet Agents browse the web?

CivikNet Agents can combine your internal knowledge with anything from the public web. As a user, you can control the scope for each question. As an administrator, you can enable or disable web browsing for any assistant you create in CivikNet Agents.

Which types of data can CivikNet Agents handle?

CivikNet Agents excels in reading documents (e.g. PDF, Powerpoint, Word, Docs), structured files (e.g. Excel, Sheets, CSV), and videos (e.g. MP4). CivikNet also connects to third-party systems such as CRMs with additional data types. Additional data types are added continuously.

How does CivikNet Agents store my data?

Data added through integrations and/or through upload to CivikNet Agents is indexed and stored in a cloud instance. CivikNet Agents performs logical data isolation to keep customer data isolated. Enterprise organizations can get their own single-tenant deployment. Data at rest is encrypted with AES 256 and data in transit is encrypted with TLS 1.2+.

How are data permissions handled?

There are three ways documents can be added to CivikNet Agents, (i) through direct upload, (ii) through a private integration, and (iii) with a shared integration. The accessibility depends on how the document has been added and what admin settings are configured.

For direct uploads, access management is handled directly in CivikNet Agents through individual document access management or through collections. Admins can set up which users can do what.

For private integrations, each user is self-authenticating and can see results only from the scope they have access to. For example, a user can see the same results from Google Drive that they could see if searching directly from their authenticated Google Drive.

For shared integrations, admins can control which documents should be available to all users who have access to a collection and/or the entire workspace.

Which LLMs does CivikNet Agents use? Are third-party models trained on my data?

CivikNet Agents’s architecture is built agnostic to the underlying large language models. Enterprise users can select between a range of LLM models and providers. CivikNet’s default model options are not trained on Content Data. We utilize enterprise security arrangements and, whenever possible, a Zero-Day Retention (ZDR) policy with our third parties. For specific LLM requirements, please get in touch with us.

Which languages does CivikNet Agents support?

The assistant can speak more than 50 languages. For specific inquiries, please get in touch with us.
